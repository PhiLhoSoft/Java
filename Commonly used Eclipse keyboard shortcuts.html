Commonly used Eclipse (keyboard) shortcuts

Edition

Ctrl+Shift+O
Organize imports

Imports unknown classes, automatically if unique, showing a choice dialog if it is ambiguous (eg. java.util.List vs. java.awt.List)

See Window > Preferences > Java > Code Style > Organize Imports if you want to define the import orders (eg. put your own classes at the end of the list).

See Window > Preferences > Java > Appearance > Type Filters to define what imports you will never use, like java.awt.*... This way, List will be imported automatically from java.util (unless it is found in another library). Note: for GWT dev, we also add com.google.gwt.dev.* and com.google.gwt.thirdparty.*

See Window > Preferences > Java > Editor > Content Assist > Favorites to define what static imports are proposed . I added org.mockito.Matchers.* and org.mockito.Mockito.* to support easy test writing. We can add org.fest.assertions.Assertions.* too. Note that JUnit assertions are part of this list by default (not displayed but given). Note that Ctrl+Shift+O doesn't add static imports, so you have to hover over the reported error (eg. mock underlined in red), and choose (in this case) Add static import for 'Mockito.mock'.

Ctrl+Shift+F
Reformat the current buffer (by default) or the selected lines.

I personally prefer to limit the scope of the formatting to avoid messing up manual formatting (line breaks and such).

Ctrl+I
Reindent the selected lines. Context sensitive, so select whole blocks!
Ctrl+Alt+A
Toggle the rectangular selection mode.

In this mode, selection (with mouse or shifted arrow keys) acts on rectangular areas. Delete will remove all chars inside the rectangle, typing will replace the selection with the typed characters on all lines, pasting acts on all lines, etc.

Alt+Up / Alt+Down
Move line where the cursor is, or selected lines, up or down.
Alt+Left / Alt+Right
Back / Forward to previously visited locations.

Navigation

Click on the return type of a method
to highlight the exit points of the method, including exceptions. Need Mark Occurrences to be active (Alt+Shift+O to toggle.)
Ctrl+hover
Over a class (eg. in a variable definition): shows class definition.

Over a method call: offers to open the declaration, the implementation, the return type (if any).

Over a field, a parameter or a variable: offers to open the declaration or the declared type.

F3
Go to class / field definition.
Ctrl+T
Quick Type Hierarchy. View and navigate to super- or sub-types. On a method call, allows to go to the interface definition or the implementations.

Note: hitting Ctrl+T again when the hierarchy is displayed allows to view the implemented interfaces.

Ctrl+Alt+H
On a method. Shows where this method is called. You can navigate to the callers of the callers, etc.
Ctrl+Shift+G
Find References. Shows all the places where the type is used.

Refactoring

Ctrl+Alt+R
Rename.
Alt+Shift+C
Change Method Signature. Shows a dialog allowing to change, on definition and call sites, the signature of a method.
Ctrl+Shift+1
Quick Fix. Proposes solutions about the selected item, varying depending on context. For example:
  • On a parameter of a method definition, allows to assign the parameter to an existing field (same type) or to a new field (it will create it).
  • On a field, proposes to create the getters and setters.
  • On a variable, allows to inline it.
  • On a selection in the parameter list of a method call, allows to assign to a new variable (reverse of inline).
  • Etc.
Ctrl+Alt+M
Extract (selected lines) to method

Debugging

Double-click on the margin
Toggle a breakpoint on this line.
F5
Step into the next method call.
F6
Step over, go to the next line in the same method.
F7
Step out, run to the end of the method and stop after the call site.
F8
Continue, run until the next breakpoint is meet or until the program ends.
Ctrl+R
Run until the selected line. Equivalent to setting a temporary breakpoint on the current line, continue the run up to this point, stop there and remove the breakpoint.
Ctrl+Alt+Click
on a method name: step into the method, stepping over the method calls in the parameter list. If we have add(getX(), getY()) for example, Ctrl+Alt+Click will step into add(), stepping over the getX() and getY() calls.
Ctrl+Shift+I
Inspect variable.
Drop to frame (icon on toolbar)
By default, no shortcut for this one. Move the instruction pointer to the start of the current method, to be able to run it again, starting over.
